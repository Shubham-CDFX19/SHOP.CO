Contributing to SHOP.CO
======================

Thanks for your interest in contributing! To keep contributions easy to review and maintain, please follow these guidelines:

- Fork the repository and create a feature branch: `git checkout -b feature/your-feature`
- Open a clear PR describing the change and linking any relevant issue.
- Keep commits small and descriptive. Use conventional commit messages if possible (e.g., `feat:`, `fix:`, `chore:`).
- Add tests for new behavior where applicable.
- Don't include secrets or .env files in commits.

Code of Conduct
---------------
Be respectful, write helpful PR descriptions, and respond to review comments constructively.
